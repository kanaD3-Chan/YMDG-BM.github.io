副板（Blue Pill，STM32F103C8）负责接收主板（ESP32-S3）通过 UART 发来的电机控制指令，驱动一个步进电机（调节靠背角度）和两个直流电机（驱动轮子）。

<!-- more -->

## 硬件接线

| 功能 | 引脚 | 说明 |
|---|---|---|
| USART1 TX | PA9 | 接 TTL 模块 RX |
| USART1 RX | PA10 | 接 TTL 模块 TX |
| 步进电机 IN1~IN4 | PA0~PA3 | ULN2003 驱动板 |
| 左轮 AIN1 / AIN2 | PB14 / PB15 | TB6612 Motor A |
| 右轮 BIN1 / BIN2 | PB13 / PB12 | TB6612 Motor B |
| 状态 LED | PC13 | 板载 LED（低电平亮） |

::: warning 接线顺序
TTL 模块的 **TX** 接 PA10（STM32 RX），**RX** 接 PA9（STM32 TX）。接反了 TX 能工作但 RX 完全静默，不会有任何报错。
:::

## 通信协议

帧格式（6 字节）：

```
S  T  flag  cs  E  D
```

- `flag`：电机控制字节，bit[7:6] 步进、bit[5:4] 左轮、bit[3:2] 右轮，每组 `10`=正转、`01`=反转、`00/11`=停止
- `cs`：校验字节，`cs = b'S' ^ b'T' ^ flag`（XOR 后为 0 即合法）
- 回复：校验通过回 `OK`，失败回 `Err`并停止所有电机以确保安全

flag 构造示例：

```python
STEP_FWD  = 0b10 << 6   # 步进正转
LEFT_FWD  = 0b10 << 4   # 左轮正转
RIGHT_FWD = 0b10 << 2   # 右轮正转

def frame(flag):
    cs = ord("S") ^ ord("T") ^ flag
    return bytes([ord("S"), ord("T"), flag, cs, ord("E"), ord("D")])
```

## 固件架构

```
src/
├── main.rs          # 初始化 + 主循环
├── protocol.rs      # UART 接收 + 协议状态机
└── motor/
    ├── mod.rs       # Motors 聚合 + MotorCommand
    ├── uln2003.rs   # 步进电机（4相全步）
    └── tb6612.rs    # 直流电机 H 桥
```

主循环：

```rust
loop {
    match protocol.next_command() {
        Ok(cmd)  => motors.apply(&cmd),
        Err(())  => motors.stop_all(),   // 校验失败立即停机
    }
}
```

## 协议解析器

用字节级状态机，每收到一个字节推进一次状态：

@startuml
hide empty description
[*] --> Idle
Idle --> WaitingByte2 : 'S'
WaitingByte2 --> WaitingFlag : 'T'\n(init checksum)
WaitingByte2 --> Idle : other
WaitingFlag --> WaitingChecksum : any\n(store flag, xor)
WaitingChecksum --> WaitingEndE : any\n(xor cs)
WaitingEndE --> WaitingEndD : 'E'
WaitingEndE --> Idle : other
WaitingEndD --> Idle : 'D' + cs==0\n→ OK + emit cmd
WaitingEndD --> Idle : 'D' + cs≠0\n→ Err + stop
WaitingEndD --> Idle : other
@enduml

USART 直接操作 PAC 寄存器（不用 HAL Serial），原因见踩坑记录。

## 电机驱动

**步进电机（ULN2003 + 28BYJ-48）**

4 相全步序列：

| 步 | IN1 | IN2 | IN3 | IN4 |
|---|---|---|---|---|
| 0 | 1 | 1 | 0 | 0 |
| 1 | 0 | 1 | 1 | 0 |
| 2 | 0 | 0 | 1 | 1 |
| 3 | 1 | 0 | 0 | 1 |

Stop 指令直接断电（`release()`）。28BYJ-48 内部蜗轮蜗杆减速比 64:1，断电后靠背不会自行滑动。

**直流电机（TB6612）**

| 状态 | IN1 | IN2 |
|---|---|---|
| 正转 | H | L |
| 反转 | L | H |
| 滑行 | L | L |

Stop 指令调用 `coast()`（两路低电平，自由滑行）。

## 踩坑记录

**HAL Serial 的 USART1 时钟问题**

`stm32f1xx-hal 0.11` 的 `Serial::new()` 内部通过 bit-band 操作使能 USART1 时钟，在部分 Blue Pill 上实测 APB2ENR bit14 不会被置位（OpenOCD 读出 `APB2ENR = 0x401C`，bit14=0）。TX 完全没有信号输出。

解决方法：在创建 Serial 之前手动写寄存器：

```rust
rcc.apb2enr().modify(|_, w| w.usart1en().set_bit());
```

最终改为直接操作 PAC 寄存器初始化 USART，绕开 HAL Serial，问题彻底消失。

**BRR 计算**

PCLK2 = 64 MHz，目标波特率 115200：

```
BRR = round(64_000_000 / 115_200) = 556 = 0x022C
```

**`block!(rx.read()).unwrap()` 在 Overrun 时 panic**

USART 接收溢出（ORE）时 HAL 返回 `Err(nb::Error::Other(...))`，`unwrap()` 直接 panic。改用 match 处理：

```rust
match self.usart.sr().read() {
    sr if sr.ore().bit_is_set() || sr.fe().bit_is_set() => {
        let _ = self.usart.dr().read(); // 清除错误标志
    }
    sr if sr.rxne().bit_is_set() => { /* 处理字节 */ }
    _ => {}
}
```
